Comparative analysis in product prices of ecommerce companies

In simple terms this is actually a web scrapping project that takes the price information from various ecommerce sites and compares them.

The ETL process I am using to extract the data:

  • EXTRACT - Via Python Selectolax I crawl the selected category pages to extract the names and prices of the present products.
  • TRANSFORM - After I get the data I use various methods and self made functions to clean the name of the product, extract the brand and the model (which is necessary for joining the tables).
  • LOAD - because this is a bigger project that is scrapping the data of several sites every 2 days (using the windows task scheduler) I was in need of a Database that can process the data. So I created a PostgreSQL Database and a connection with Python via the psycopg2, pandas and sqlalchemy libraries. I store every shop separately in 2 tables. The first table stores the current data that is afterwards merged with the other tables to get the master table (shown below). So the first table replaces itself after every execution, while the second table appends the new data to it. So after picking the "main" table I join all the First tables using "left joins" and the cleaned up product model. The final table is shown down below. The second table will accumulate the data where it will be then possible to see the price movments.
